# feishu-skill · 飞书笔记整理 + 会议/录音纪要 + 画板可视化工作流

一个 Claude Code / Claude Agent Skill，接受三种输入、统一输出结构化的飞书文档：

- **文档 / 笔记** — 读取一份飞书笔记/文档，先把正文里嵌的图片（多为课件截图）用视觉读懂、按类型转成原生可编辑内容（课件图按「知识点→案例」提取成文字、流程图转画板、表格截图转表格、代码截图转代码块，放回原位替换原图，最终不留任何静态图）；再把全文整理成结构化的新文档（原文档只读不改、另存一份、文字逐字保真、顺序默认不变），开头新增归类总结 + 总览画板，能可视化的内容用飞书原生可编辑画板（whiteboard）画出来。
- **会议 / 录音** — 输入一场飞书会议 / 妙记，或一段普通录音（本地音视频文件），自动拿到转写文本后切分议题、提炼「讨论 / 结论 / 待办 action items / 决议」，输出一份带导读区、待办汇总表和总览画板的结构化会议纪要。录音的语音转写交给飞书妙记，无需本地 whisper / ffmpeg。

## 适用场景

- “帮我整理这篇飞书笔记，能可视化的用画板画出来”
- “把这份文档归纳一下，另存一份新的，别动原文”
- “这篇笔记里有课件图片，先把图里的内容提取成文字再整理”
- “把这场会议整理成纪要”（飞书会议 / 妙记链接）
- “这段录音帮我整理成会议纪要”（手机随手录、线下访谈、电话沟通等）
- “把会议里的待办拉出来，谁负责什么、什么时候交”

## 依赖

- `lark-cli`（飞书命令行工具）
- 配套 skill：`lark-shared`（认证/权限）、`lark-doc`（读写文档）、`lark-whiteboard`（画板）
- 会议 / 录音分支另需：`lark-minutes`（妙记：录音上传转写、取逐字稿）、`lark-vc`（会议纪要/待办/逐字稿）；可选 `lark-im`、`lark-task`（纪要发群 / 待办建任务）

## 安装

把整个目录放到你的 skills 目录下（例如 `~/.claude/skills/feishu-skill/`）即可。

## 文件

- `SKILL.md` — skill 定义与完整工作流（铁律、步骤）。
